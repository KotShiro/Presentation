In this post I would like to tell a short history of such fundamental things for understanding the WEB as WWW, HTML and CSS.

So let's get started.

how did it all start?

The world's first Internet site was presented to the world in August 1991 by British scientist Timothy Berners-Lee at http://info.cern.ch/, now the site is archived.

who started?

Timothy John Berners-Lee
from the UK. He was born on June 8, 1955 in London.

Global Web project. WWW

The place of work that played a decisive role in the fate of Tim, and indeed of all mankind, is the European Laboratory for Nuclear Research (CERN), located in Geneva (Switzerland). There, Berners-Lee developed the Inquire program, which used the method of random associations.

The main goal of Tim's program was to publish hypertext documents that could be referenced. This simplifies the search for information, its organization and storage. The project was originally planned to be implemented on the CERN intranet for local research needs in alternative data stores. At the same time, data download and access to them were possible from any computer connected to the WWW.

Work on the project lasted from 1991 to 1993. In particular, the first versions of the URL protocols (as a special case of URIs), HTTP and HTML were already proposed at that time. Also introduced was the first hypertext-based World Wide Web browser and WYSIWYG editor.

One of the fixes for the WWW has always been node decentralization. Like the ancestors of the Internet (ARPANET and NSFNet), it provides reliable operation, without geographical boundaries and barriers.

HTML

As I said earlier, HTML (Hypertext Markup Language) is intended for marking and formatting documents generated while working on the Global Web project.

HTML is the successor to SGML, but it was created to be used by non-typesetters. Thus, from the early days, HTML has the following benefits:

  • Simplicity - due to a small set of structural elements - descriptors (they are also "tags"). All tags are written in angle brackets, for example. <..tag..> and carry some kind of semantic load
  • Ability to format a document without reference to display means (be it a computer monitor, a phone screen or a reader)

1993 - HTML 1.2

In this version, out of more than forty tags, 3 tags have appeared, which hint at some kind of visual design of the document (for example, bold italic). The rest of the tags were used exclusively for logical markup.

1994 - Founding of W3C

Sir Tim Berners-Lee founded the World Wide Web Consortium (World Wide Web Consortium, W3C). The mission of the W3C was and remains to "unleash the full potential of the World Wide Web by establishing protocols and principles for the long-term development of the network." It's a huge credit to these guys that HTML was released with one basic set of tags and attributes, and web pages became what we know today.

1995 - Version 2.0

The process and approval of the new version was very leisurely, the only noticeable improvement in the new version: Queries: For example, keyword searches. Forms for transferring data from computer to server: enter your date of birth or select one of the options in the questionnaire.

1995 - Beginning of work on HTML 3.0.

The first version of the standard included many interesting things:

  • tags for creating tables,
  • markup of mathematical formulas,
  • text wrapping, etc.

The authors suddenly remembered that, in theory, HTML should only mark up the document and should not contain parameters for the graphic styles of displaying elements in the browser. From what followed the decision to add support for a new tool served to design web pages.

December 1996 - CSS

CSS (Cascading Style Sheets). Representing views of parts of a document.

So the CSS system:

  • does not formally depend on HTML,
  • has its own non-HTML syntax,
  • It is not bound by the ideological limitations of HTML,
  • Allows you to set the appearance options for any HTML tag.

With the CSS web page builder, you can finally change the font and size of anything, and, importantly, this is not made possible by introducing new tags, but by using a mechanism that affects the interpretation of already used HTML tags. ...

1997 - HTML 3.2

Version 3.2 came out a month after CSS was approved and has already been fully adapted for interacting with stylesheets.

It omits many of the 3.0 features, but adds non-standard elements supported by Netscape Navigator and Internet Explorer 3.

1997 - HTML 4.0

In this version, elements from previous versions have been pretty much cleaned up. Many tags have been recommended as deprecated and deprecated. Instead, it was necessary to use CSS style sheets. The new version included support for frames, scripts, general support for various objects.

1999 - HTML 4.01

In this version, we slightly tweaked objects, shapes and images, fixed bugs and generally created a more stable version using web developers for over 10 years.

October 28, 2014 - HTML5

From that day on, the W3C officially recommends using HTML5.

  • The new version has made the syntax stricter than the previous one,
  • Improved support for multimedia technologies
  • Added 28 new structural elements to make the code more understandable
  • Some more obsolete tags have been removed
  • More attention is paid to supporting scripts, for example javascript

At the moment, HTML development continues ...
